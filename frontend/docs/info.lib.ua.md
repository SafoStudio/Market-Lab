# TanStack Query (React Query)
### TanStack Query - бібліотека для управління серверним станом, яка спрощує отримання, кешування, синхронізацію та оновлення даних з API.

## Управління API-даними:

Товари, замовлення, користувачі, категорії

Автоматичне кешування та ревалідація

Фонове оновлення даних

## Оптимізація продуктивності:

Мінімізація зайвих запитів до бекенду

Оптимістичні оновлення UI

Пагінація та нескінченна прокрутка

## Обробка помилок:

Автоматичні повторні спроби

Стан завантаження та помилок

Інвалідація кешу

Конфігурація в проекті:
QueryProvider в корені додатка

Глобальні налаштування кешування

Кастомні хуки для кожного типу даних




# Zustand
### Zustand - мінімалістична бібліотека для управління клієнтським станом, альтернатива Redux або Context API.

## Основні завдання в проекті:
Клієнтський стан:

UI стан (тема, модальні вікна, sidebar)

Стан кошика покупок

Стан аутентифікації

## Глобальний доступ:

Дані доступні у всьому додатку

Незалежність від серверних даних

Проста інтеграція з компонентами

## Middleware:

Persist (збереження в localStorage)

DevTools (налагодження)

Immer (незмінні оновлення)

## Структура сторів:
authStore - аутентифікація та користувач

cartStore - кошик покупок

uiStore - інтерфейсні налаштування




# Взаємодія бібліотек TanStack & Zustand
Розподіл відповідальностей:
```
┌─────────────────────────────────────────┐
│          TanStack Query                 │ ← Серверний стан
│    • API дані (товари, замовлення)      │
│    • Кешування та синхронізація         │
│    • Фонові оновлення                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│            Zustand                      │ ← Клієнтський стан
│    • UI стан (тема, модалки)            │
│    • Локальні дані (кошик)              │
│    • Сесія користувача                  │
└─────────────────────────────────────────┘
```

## Переваги комбінації:
Чітке розділення - серверні vs клієнтські дані

Оптимізація продуктивності - менше ререндерів

Покращена підтримка - легше тестувати та дебажити

Гнучкість - можна замінити одну бібліотеку без впливу на іншу

## Ключові особливості архітектури
Інтеграція з Next.js:
Серверний рендеринг (SSR) з TanStack Query

Гідратація клієнтського стану

Оптимізація для SEO

## Безпека:
Токени зберігаються в HTTP-only cookies

Автоматичне оновлення токенів

Захист від CSRF-атак

## Масштабованість:
Модульна структура сторів

Легке додавання нових endpoint'ів

Підтримка TypeScript




# Tailwind CSS 
### Tailwind CSS - утилітарний CSS-фреймворк для швидкої розробки інтерфейсів з готовими класами.

## Швидкість розробки:

Готові утилітарні класи

Консистентний дизайн-система

Швидке прототипування

## Кастомізація:

Власна дизайн-система (кольори, шрифти, відступи)

Темна/світла тема

Адаптивний дизайн

## Оптимізація:

Tree-shaking (видалення невикористаних класів)

Purge CSS у продакшені

Мінімальний bundle розмір

```
┌─────────────────────────────────────────┐
│        Глобальні стилі                  │
│    • app/globals.css                    │
│    • Tailwind директиви                 │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│        Компонентні стилі                │
│    • Inline Tailwind класи              │
│    • CVA для варіантів                  │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│        Кастомні утиліти                 │
│    • Додаткові класи в config           │
│    • CSS змінні для тем                 │
└─────────────────────────────────────────┘
```
В проекті використовуються tailwind-merge та clsx:

tailwind-merge - для об'єднання Tailwind класів без конфліктів (останній клас має пріоритет)

clsx - для зручного формування умовних CSS класів з булевою логікою

Обидві бібліотеки спрощують роботу з динамічними класами в компонентах.
